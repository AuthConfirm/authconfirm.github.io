# AuthConfirm Docs

Welcome to the documentation for **AuthConfirm**, the standard fallback authentication method for [Scratch JP](https://scratchjp.org/)!

## Getting Started

To get started with AuthConfirm, you need to understand how the manual authentication works. Because Scratch cloud variables can sometimes be unusable, AuthConfirm offers a manual, comment-based confirmation as a fallback.

### Authentication

Here is the standard process for a user authenticating:
1. The user asks one of the moderators to authenticate them through ModMail.
2. The moderator tells a code to the user.
3. The user goes to [the AuthConfirm Scratch project](https://scratch.mit.edu/projects/1275511960) and pastes the code into the project's comment section.
4. The moderator checks the comment section to verify the user.

## Notes

### Availability
AuthConfirm is only available if ScratchAuther is unavailable.

### Delays
Since AuthConfirm is done manually, there can be delays in the authentication process. Please be patient with the moderators as they try to authenticate you.

### Authenticating New Scratchers
New Scratchers may not be able to authenticate using AuthConfirm. For more information, see [よくある質問](https://discord.com/channels/888638910589206569/1321412931669331998/1321414097161555989) in the [#┃お問い合わせ](https://discord.com/channels/888638910589206569/1321412931669331998) channel.
